## Directory Purposes

### `src/app/`
- **Purpose:** Next.js App Router - pages and API routes
- **Contains:** Page components, layouts, API route handlers
- **Key files:**
  - `layout.tsx` - Root layout with providers
  - `page.tsx` - Landing page (redirects to dashboard)
  - `login/page.tsx` - Login form

### `src/app/api/`
- **Purpose:** REST API endpoints
- **Contains:** Route handlers for CRUD operations
- **Key files:**
  - `variants/route.ts` - Product variants CRUD
  - `raw-materials/route.ts` - Raw materials CRUD
  - `tasks/route.ts` - Task management
  - `employees/route.ts` - Employee management
  - `transactions/route.ts` - Finance transactions
  - `batches/route.ts` - Production batches

### `src/components/`
- **Purpose:** Shared React components
- **Contains:** UI components, feature components, dialogs
- **Key files:**
  - `providers.tsx` - React Query + Auth providers
  - `sidebar-wrapper.tsx` - Sidebar with mobile support
  - `app-sidebar.tsx` - Navigation sidebar
  - `toast-container.tsx`, `toast.tsx` - Notification system

### `src/components/ui/`
- **Purpose:** shadcn/ui base components
- **Contains:** Button, Input, Dialog, Sheet, etc.
- **Pattern:** Generated by shadcn, imported from `@/components/ui/*`

### `src/contexts/`
- **Purpose:** React contexts for global state
- **Contains:** Auth context (authentication state)
- **Key files:** `auth-context.tsx`

### `src/hooks/`
- **Purpose:** Custom React hooks
- **Contains:** Hooks like use-mobile
- **Key files:** `use-mobile.ts`

### `src/lib/`
- **Purpose:** Core utilities and clients
- **Contains:** Prisma client, Supabase client, auth helper, stores
- **Key files:**
  - `prisma.ts` - Database client
  - `supabase.ts` - Supabase browser client
  - `auth-helper.ts` - Server-side auth utilities
  - `utils.ts` - cn() utility for className merging
  - `stores/ui.ts` - Zustand UI store

### `prisma/`
- **Purpose:** Database schema and migrations
- **Contains:** Prisma schema definition
- **Key files:** `schema.prisma`

## Naming Conventions

### Files
- **Pages/Components:** PascalCase (`DashboardPage.tsx`, `TaskCard.tsx`)
- **Utilities/Lib:** kebab-case (`auth-helper.ts`, `use-mobile.ts`)
- **API Routes:** kebab-case (`variants/route.ts`)
- **Config:** camelCase/snake_case based on tool (`next.config.ts`, `eslint.config.mjs`)

### Directories
- **Pages:** kebab-case (`raw-materials/`, `tasks/`)
- **Components:** kebab-case for folder (`ui/`, `task-board.tsx` directly)
- **Lib:** kebab-case (`stores/`, `utils.ts`)

### Variables/Functions
- camelCase: `getCurrentUser`, `useAuth`, `activeModal`
- PascalCase: Components, Types, Interfaces

### Database Models (Prisma)
- PascalCase model names: `products`, `product_variants`, `raw_materials`
- camelCase field names: `productId`, `createdAt`, `isActive`

## Where to Add New Code

### New Feature (Page + API)
1. **Page:** Create `src/app/[module]/page.tsx`
2. **API:** Create `src/app/api/[module]/route.ts`
3. **Sidebar:** Update `src/components/app-sidebar.tsx` navigation

### New Component
1. **Shared:** Add to `src/components/` or `src/components/ui/`
2. **Page-specific:** Add alongside page in same directory

### New Utility
1. **Client library:** Add to `src/lib/`
2. **Store:** Add to `src/lib/stores/` (Zustand)
3. **Context:** Add to `src/contexts/`

### New Database Table
1. **Schema:** Add model to `prisma/schema.prisma`
2. **Client:** Run `npx prisma generate`
3. **API:** Add route in `src/app/api/`
