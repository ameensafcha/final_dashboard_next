Current issue: Middleware runs in Edge Runtime and fails when importing Prisma client.

Research findings:
- Next.js middleware runs at Edge by default
- Edge Runtime doesn't support Node.js crypto module  
- @prisma/adapter-pg uses pg driver which requires Node.js crypto
- Solution: Set middleware to use Node.js runtime: `export const config = { runtime: 'nodejs' }`

Reference: Prisma docs confirm "node-postgres (pg) is currently not supported on Vercel Edge Functions"
