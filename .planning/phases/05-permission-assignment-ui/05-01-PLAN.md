Fix the GET handler in src/app/api/roles/permissions/route.ts:

The current implementation uses:
```typescript
const permissions = await prisma.role_permissions.findMany({
  distinct: ['permission'],
  select: { permission: true }
});
```

This returns only permissions that are ASSIGNED to at least one role. It should return ALL available permissions.

Change the GET handler to:
1. Query all DISTINCT permission strings from the role_permissions table
2. Return them as an array of strings (not objects)
3. This gives the UI the complete list of permissions that exist in the system

The fix:
```typescript
export async function GET() {
  try {
    const authCheck = await requireAdminApi();
    if (authCheck instanceof NextResponse) return authCheck;
    
    // Get ALL distinct permissions that exist in the DB
    const permissions = await prisma.role_permissions.findMany({
      distinct: ['permission'],
      select: { permission: true },
      orderBy: { permission: 'asc' }
    });
    
    return NextResponse.json(permissions.map(p => p.permission));
  } catch (error) {
    console.error('Permission Error:', error);
    return NextResponse.json({ error: 'Failed to fetch permissions' }, { status: 500 });
  }
}
```

Note: If no permissions exist in DB yet, return the default 6 permissions as fallback so the UI still works.
