**Golden Rule:** Stop, Think, Plan, and Ask before you Code.

## 1. Plan Before You Act 🧠
- **No Blind Execution:** Do not jump straight into writing code or executing tasks.
- **Formulate a Plan:** Analyze the user's request thoroughly and create a step-by-step execution plan first.

## 2. Ask Questions First ❓
- **Identify Ambiguities:** Look for missing information, edge cases, or unclear requirements.
- **Consult the Developer:** If there is any doubt, pause and ask the developer specific questions. Do not assume anything.

## 3. Code Only When 100% Clear ✅
- **Zero Ambiguity:** Proceed to writing code *only* when all questions have been answered and the logical flow is entirely clear.
- **Execute with Purpose:** Every line of code written must have a clear intention backed by the approved plan.

## 4. Research When Stuck 🔍
- **No Guessing:** If you do not understand a concept, API, error, or library, do not hallucinate or guess.
- **Deep Dive:** Take the time to research the problem, analyze documentation, and find the correct technical approach.

## 5. Leverage Your Full Skillset 🛠️
- Use every tool and skill at your disposal.
- Apply advanced reasoning, architectural best practices, debugging skills, and optimization techniques to deliver the highest quality output possible.
